# 黄金价格交易时间管理说明

## 问题背景

**用户反馈**：收盘后应该不会有波动，但之前的系统数据一直在跳动

**原因**：之前的系统使用完全随机的模拟数据，没有考虑到真实市场的交易时间限制

## 解决方案

### 交易时间定义

黄金市场的交易时间一般分为：

| 时段 | 时间 | 特征 |
|------|------|------|
| **交易时段** | 09:00 - 21:00 | 市场活跃，价格波动 |
| **非交易时段** | 21:00 - 09:00 | 市场关闭，价格稳定 |

### 实现原理

```javascript
// 1. 获取当前小时
const hour = date.getHours();  // 返回 0-23

// 2. 判断是否交易时间
const isMarketOpen = hour >= 9 && hour < 21;

// 3. 根据交易时间决定波动
if (isMarketOpen) {
    // 交易时间（09:00-21:00）：有正常波动
    const randomWalk = (Math.random() - 0.5) * 2;
    const trend = (Math.random() - 0.5) * 0.3;
    volatility = randomWalk + trend;
    price = price + volatility;
} else {
    // 非交易时间（21:00-09:00）：无波动，保持前一个价格
    volatility = 0;
    if (data.length > 0) {
        price = data[data.length - 1].close;  // 保持前一个收盘价
    }
}
```

**交易时段（09:00-21:00）的蜡烛**：
```
    ▲ 高 (High)
    │
  ┌─┼─┐  ← 蜡烛体（Body）清晰可见
  │ │ │  ← 开盘和收盘价有差异
  └─┼─┘
    │
    ▼ 低 (Low)
```
- 蜡烛体清晰可见
- 开盘≠收盘
- 有影线（wicks）

**非交易时段（21:00-09:00）的蜡烛**：
```
  ─────  ← 蜡烛体很小或不存在
         ← 开盘=收盘
```
- 蜡烛体很小（或为线）
- 开盘≈收盘
- 影线很短

## 代码查看

```javascript
// 生成历史数据函数中的关键部分
for (let i = config.points; i >= 0; i--) {
    const date = new Date(timestamp);
    const hour = date.getHours();  // ← 获取小时
    
    // ← 判断交易时间
    const isMarketOpen = hour >= 9 && hour < 21;
    
    let volatility = 0;
    if (isMarketOpen) {
        // ← 交易时间：有波动
        const randomWalk = (Math.random() - 0.5) * 2;
        const trend = (Math.random() - 0.5) * 0.3;
        volatility = randomWalk + trend;
    } else {
        // ← 非交易时间：无波动
        volatility = 0;
    }
    
    // 生成OHLC数据
    data.push({
        date: date,
        open: parseFloat(open.toFixed(2)),
        high: parseFloat(high.toFixed(2)),
        low: parseFloat(low.toFixed(2)),
        close: parseFloat(close.toFixed(2)),
        isMarketOpen: isMarketOpen  // ← 记录交易状态
    });
}
```

## 常见问题

### Q1: 为什么非交易时段的蜡烛很小？
**A**: 这是正确的行为。非交易时段市场关闭，价格不会变动，所以开盘≈收盘，蜡烛体很小。

### Q2: 我能看到具体的波动值吗？
**A**: 是的，鼠标悬停在任何蜡烛上都能看到详细的开盘、最高、最低、收盘价格。

### Q3: 为什么交易时段和非交易时段有明显的区别？
**A**: 这模拟了真实市场情况。交易时间有交易者活动导致价格波动，非交易时间市场关闭，价格保持上一个收盘价。

### Q4: API是否会自动每小时更新一次？
**A**: 当前版本需要用户手动切换时间框架来触发数据更新。真实部署时建议加入自动刷新机制。

## 改进方向

1. **更复杂的波动模型**
   - 当前使用随机游走
   - 可改进为基于真实交易量的模型

2. **自动刷新**
   - 每小时自动获取新数据
   - WebSocket实时推送

3. **更细的时间粒度**
   - 当前为小时粒度
   - 可改进为分钟粒度（需要更多API支持）

4. **多市场支持**
   - 加入亚市、欧市、美市数据
   - 显示不同市场的交易时间
